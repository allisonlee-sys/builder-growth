KPI Framework

We own the full self-serve → expansion journey, so a single metric can't reflect performance. We operate with a KPI stack anchored on a leading indicator of long-term value.

Primary Focus Metric

Activation Rate

% of new orgs reaching first value within 7 days. If users don't get here fast, nothing downstream matters.

02
Current Constraints

Our ability to drive PLG and builder growth is currently limited. We will prioritize clarity, focus, and coordination before expanding scope.

Fragmented Ownership

Key surfaces like homepage and lifecycle lack clear owners — duplicated work, slow progress.

No Shared Cadence

Lack of cross-pod visibility makes prioritization and dividing work difficult.

Unclear Builder Segment

Without a clear definition, targeting, personalization, and measurement suffer.

Team Capacity & Ramp

Requires focus on a small number of high-impact initiatives until capacity grows.

03
30 / 60 / 90 Plan

We will focus on a small number of high-impact areas at a time given team size and current constraints.

Clarity + Momentum

  • Target 1–2 high-impact moments (e.g. early drop-off, lack of return usage)

04
How We Operate
01
Own Outcomes, Not Execution
Accountability
Accountable for whether the PLG motion works. If activation, retention, or conversion doesn't move, we treat it as our problem to solve.
02
Operate Across the Full Funnel
Scope
Acquisition → activation → habit → monetization → expansion. We don't optimize one stage at the expense of the system.
03
Run End-to-End
Speed
When ownership is unclear, we own it fully: hypothesis, copy, build coordination, measurement, results. We partner closely, but we do not wait.
04
Lead With Experimentation
Mindset
Messaging is a hypothesis. Every surface is continuously iterated. We ship and learn. If it doesn't convert, we change it.
05
Translate Narrative Into Performance
Craft
PMM defines the narrative. We make it work in-market: simplify, adapt for the internet, test, iterate. If messaging doesn't convert, it's refined until it does.
06
Build Systems, Not One-Offs
Scale
Repeatable systems across onboarding, lifecycle, in-product messaging, monetization flows, and PLS signal generation. If it works once, we scale it.
07
Focus on What Moves Metrics
Ruthless
Clear owner. Measurable goal. Defined timeline. We go deep on fewer, higher-impact initiatives.
08
Close Gaps in Ownership
Drive
Where ownership is fragmented, we identify the gap, run experiments, and scale what works. We are not blocked by ambiguity.
05
Stop / Start / Continue
Stop
  • Waiting for perfect inputs before moving
  • Measuring success by outputs instead of outcomes
  • Treating messaging as fixed
  • Optimizing isolated funnel stages
Start
  • Owning the activation → habit gap
  • Running end-to-end experiments
  • Building builder-specific journeys
  • Translating narrative into performance
  • Establishing a consistent test cadence
  • Instrumenting builder identification & activation quality
Continue
  • Iterating on pricing & monetization surfaces
  • Developing PLS signals and routing
  • Partnering cross-functionally while maintaining outcome ownership
Operating Principle
If something is not working, we assume the system is broken and fix it.
